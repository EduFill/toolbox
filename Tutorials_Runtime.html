ROS - Robot Operating System

In order to use the EduFill software toolbox, the robot software framework ROS has to be installed. The version which was verified with our components is "electric". Installation instructions for "ros-electric-desktop-full" are on:

ROS electric installation instruction

Additionally, several other packages have to be installed. For this run in an Ubuntu system:

sudo apt-get install ros-electric-desktop-full ros-electric-arm-navigation ros-electric-pr2-controllers ros-electric-object-manipulation ros-electric-pr2-kinematics ros-electric-joystick-drivers ros-electric-laser-drivers ros-electric-cob-common ros-electric-pr2-simulator ros-electric-openni-kinect ros-electric-pr2-apps ros-electric-bosch-drivers python-pygraphviz libmysqlclient-dev python-scipy libcap-dev bzr yaml-cpp0.2.6-dev ros-electric-orocos-toolchain ros-electric-rtt-ros-integration ros-electric-rtt-ros-comm ros-electric-rtt-common-msgs ros-electric-rtt-geometry libcap2-bin python-scipy python-setuptools

Clone repositories and external software

The EduFill software toolbox is maintained in multiple git repositories, currently hosted at https://mas.b-it-center.de/git/projects/edufill. These repositories will soon move to the new project homepage currently under construction at www.edufill.org.

Firstly, a folder where all components will be stored has to be created.Open terminal and execute the command: 

mkdir ros_stacks

Then Edufill components should be cloned:

  • Create Edufill folder: 

cd ros_stacks
mkdir Edufill
cd edufill

  • Clone repository: 

git clone http://mas.b-it-center.de/git/projects/edufill/components

Note: Currently password protected while we prepare the first public beta.

Next step is get external software:

  • Create external_software folder: 

cd ros_stacks
mkdir external_software
cd external_software

  • Clone external software: 

git clone https://github.com/b-it-bots/hbrs-youbot-hackathon-common
git clone https://github.com/b-it-bots/third-party-software.git
git clone https://github.com/youbot/youbot_driver.git
git clone https://github.com/b-it-bots/youbot-ros-pkg.git
git clone https://github.com/b-it-bots/youbot-manipulation.git
git clone http://robotics.ccny.cuny.edu/git/ccny-ros-pkg/ccny_vision.git

Allow ROS access to components

In order for ROS to gets access to the cloned repository, the file ~/.bashrc has to be changed. In terminal: 

sudo nano ~/.bashrc

Several variables have to be added:

  • $ROS_PACKAGE_PATH for components: 

export ROS_PACKAGE_PATH=/home/your_user_name/ros_stacks/electric/EduFill:$ROS_PACKAGE_PATH

and for external software: 

export ROS_PACKAGE_PATH=/home/your_user_name/ros_stacks/external_software:$ROS_PACKAGE_PATH

  • ROBOT, which robot will be used: 

export ROBOT=youbot-edufill2

  • ROBOT_ENV, where robot will be operated: 

export ROBOT_ENV=brsu-home

Save file Ctrl+X and press Y. At the end apply changes by: 

source ~/.bashrc

  • Before launching environment, packages have to be rebuild: 

rosmake youbot_common youbot_drivers youbot_navigation edufill_2dnav

  • To check if all components are installed, launch: 

 roslaunch youbot_description youbot_publisher.launch

If simulation environment Gazebo starts, you will see a youBot in its environment. It indicates the installation of all components and software went fine. (Figure 1)
