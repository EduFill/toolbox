[edit] ROS - Robot Operating System

In order to operate robot software framework ROS has to installed. Version which was verify with our components is "electric". Instalation instruction for "ros-electric-desktop-full" is on :

ROS electric installation instruction

Additionally, several other packages has to be installed:

sudo apt-get install ros-electric-desktop-full ros-electric-arm-navigation ros-electric-pr2-controllers ros-electric-object-manipulation ros-electric-pr2-kinematics ros-electric-joystick-drivers ros-electric-laser-drivers ros-electric-cob-common ros-electric-pr2-simulator ros-electric-openni-kinect ros-electric-pr2-apps ros-electric-bosch-drivers python-pygraphviz libmysqlclient-dev python-scipy libcap-dev bzr yaml-cpp0.2.6-dev ros-electric-orocos-toolchain ros-electric-rtt-ros-integration ros-electric-rtt-ros-comm ros-electric-rtt-common-msgs ros-electric-rtt-geometry libcap2-bin python-scipy python-setuptools


[edit] Clone repositories and external software

Firstly, a folder where all components will be stored has to create: 

mkdir ros_stacks

Then Edufill components should be cloned:

  • Create edufill folder: 
cd ros_stacks
mkdir edufill
cd edufill
  • Clone two repositories: 
git clone http://<name>:<passwd>@mas.b-it-center.de/git/projects/edufill/components

and 

git clone http://<name>:<passwd>@mas.b-it-center.de/git/projects/edufill/toolbox

User <name> and <passwd> given by email. Next step is get external software need to make robot work:

  • Create external_software folder: 
cd ros_stacks
mkdir external_software
cd external _software
  • Clone external software: 
git clone https://github.com/b-it-bots/hbrs-youbot-hackathon-common

git clone https://github.com/b-it-bots/third-party-software.git

git clone https://github.com/youbot/youbot_driver.git

git clone https://github.com/b-it-bots/youbot-ros-pkg.git

git clone https://github.com/b-it-bots/youbot-manipulation.git

git clone http://robotics.ccny.cuny.edu/git/ccny-ros-pkg/ccny_vision.git

In order for ROS to get access to the cloned repository, the file ~/.bashrc has to be changed: 

sudo nano ~/.bashrc

And variable $ROS_PACKAGE_PATH has to added: 

export ROS_PACKAGE_PATH=/home/your_user_name/ros_stacks/electric/EduFill:$ROS_PACKAGE_PATH

and : 

export ROS_PACKAGE_PATH=/home/your_user_name/ros_stacks/external_software:$ROS_PACKAGE_PATH

and ROBOT_ENV: 

export ROBOT_ENV=brsu-home

and ROBOT: 

export ROBOT=youbot-edufill2

At the end apply changes by: 

source ~/.bashrc
  • Before launch enviromant cloned packeges has to be made: 
rosmake youbot_common youbot_drivers youbot_navigation edufill_2dnav
  • To check is all components are installed launch : 
 roslaunch youbot_description youbot_publisher.launch

If simulation environment gazebo starts and user see youbot that mean user install all (Figure 1).

Figure 1. Youbot in Gazebo simulation

[edit] Make own scenario

  • In order to start making user own scenarios go to link: 
https://mas.b-it-center.de/edufill/demo

or 

https://test.edufill.org
  • On this web page follow createApp link and Blockly workspace will appear. Blockly is a web-based, graphical programming editor, where application can be created.
  • On the web page there are three tabs: Blocks, Python and XML. The Blocks tab consists of several subtabs. In this scenario user will need only the "High Level" subtab.
  • Firstly, several modules need to be imported. It can be done using block "start application". These blocks include all import modules which user needs to make and run the script. Now user can start playing around with blocks to make the robot move, take objects and explore world and etc.
  • Our first task is to move towards an object, fetch it and carry to goal position on map (platform).
  • The user can go deeper into details of showcase and explore "High-Level" subtab, where user can find another blocks that can be drag and dropped to the workspace and connect it to "start application" block:
    • "move base to (S1)"
    • "find and Grasp (cube)"
    • "move arm to (initialpose)"
    • "move base to (EXIT)"

The "S1" is plane where object is staying and "EXIT" position where out of robot arena.They are predefine as navigation goals.

Figure 2. Blockly diagram for Fetch and Carry scenario, High Level
  • After all wanted blocks are connected in need sequence on workspace (Figure 2) , python code can be generated. Click on "Python" tab and code can be seen in new workspace. Next, these code can be stored by "Save Python" button.

(Google Chrome) The Save as window pop up. Navigate to ~/edufill/components/edufill_blockly/scripts/script_default.py file and press Save. (FireFox)It will open new tab with python code. Ctrl-A and copy this text to ~/edufill/components/edufill_blockly/scripts/script_default.py

  • In order to run simulation of the showcase several steps should be done: 
 roslaunch edufill_blockly  blockly_default.launch

Open second terminal run script: 

 cd ~/scripts/
 ./script_defult.py
  • Now user can see how robot moves and take object in gazebo simulation. On Figure 3 presented initial state of fetch and carry scenario.
Figure 3. Youbot in scenario start position, cube on S1 platform
